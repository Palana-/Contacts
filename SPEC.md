# 脸谱通讯录 - 设计方案

## 1. 项目概述

**项目名称**：脸谱通讯录
**项目类型**：跨平台移动应用（Android & iOS）
**核心功能**：以头像照片为核心的通讯录管理应用，专为不识字用户设计
**目标用户**：不识字人群、视觉化交互偏好用户

## 2. 技术方案

### 2.1 跨平台框架选择

**推荐方案：Flutter**
- 原因：原生性能、丰富的摄像头/相册插件、良好的联系人API封装
- 插件生态完善，社区活跃

**替代方案：React Native**
- 优点：JavaScript生态庞大
- 缺点：头像与系统通讯录同步（Android）实现较复杂

### 2.2 关键依赖

| 功能 | 插件 | 说明 |
|------|------|------|
| 联系人读写 | `contacts_service` + `flutter_contacts` | 读写系统通讯录 |
| 头像同步 | `image_picker` + `path_provider` | 拍照/选图，本地存储 |
| 拨号 | `url_launcher` | 调起系统拨号 |
| 来电显示 | Native Module (Android) | 需要原生开发 |
| 数据存储 | `sqflite` + `hive` | 本地数据库 |
| 导入导出 | `share_plus` + `file_picker` | 文件分享 |
| 权限管理 | `permission_handler` | 运行时权限 |

## 5. 数据架构

### 5.1 本地数据模型

```dart
class Contact {
  String id;           // 唯一标识
  String? name;        // 姓名（非必填）
  String? phone;       // 电话
  String? avatarPath;  // 本地头像路径
  String? systemId;   // 对应系统通讯录ID
  bool isFavorite;     // 是否收藏
  DateTime createdAt;  // 创建时间
  DateTime updatedAt;  // 更新时间
}
```

### 5.2 数据存储

| 数据 | 存储方式 |
|------|----------|
| 联系人数据 | Hive本地数据库 |
| 头像图片 | 应用私有目录 |
| 系统通讯录映射 | Hive + systemId关联 |

## 6. Android原生模块

### 6.1 来电显示头像

需要开发原生Android模块实现：
1. 监听来电广播（`android.intent.action.PHONE_STATE`）
2. 查询系统通讯录获取头像
3. 使用RemoteViews自定义来电界面

**注意**：此功能需要原生开发，无法纯Flutter实现

### 6.2 头像写入系统通讯录

通过`flutter_contacts`插件写入系统通讯录，系统会自动用于来电显示

## 7. 权限需求

| 权限 | 用途 | 平台 |
|------|------|------|
| READ_CONTACTS | 读取系统通讯录 | Android/iOS |
| WRITE_CONTACTS | 写入系统通讯录 | Android/iOS |
| READ_EXTERNAL_STORAGE | 读取相册 | Android |
| CAMERA | 拍照添加头像 | Android/iOS |
| CALL_PHONE | 直接拨号 | Android |
| READ_PHONE_STATE | 来电显示 | Android |

## 8. 导出文件格式

### 8.1 VCard格式（推荐）

```
BEGIN:VCARD
VERSION:3.0
FN:张三
TEL;TYPE=CELL:13800138000
PHOTO;VALUE=URI:file:///path/to/photo.jpg
END:VCARD
```

### 8.2 CSV格式

```csv
姓名,电话,头像路径
张三,13800138000,/path/to/photo.jpg
```
